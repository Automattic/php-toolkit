# Markdown in Git. WordPress in charge.

Agents work in Git on ordinary .md files. Push MD makes WordPress the remote that checks roles, creates revisions, rejects stale pushes, and renders the site.

## Markdown files

Posts and pages appear as readable .md files for agents, diffs, and local editors.

## WordPress remote

Git clients clone directly from /wp-json/git/v1/md.git. There is no second repository to reconcile.

## Existing roles

Git over HTTPS uses WordPress authentication and respects current capabilities.

## Revisions

Accepted Markdown pushes go through WordPress APIs and create WordPress revisions.

## WordPress coordinates the work.

Push MD gives every authenticated worker the same Git-shaped view of supported content. Coding agents, local editors, and WP-Admin all coordinate through WordPress, so there is no second source of truth drifting out of date. 

Agent A      -- git pull  --> WordPress
Agent B      -- git push  --> WordPress
Local editor -- Markdown  --> WordPress
WP-Admin     -- revisions --> WordPress
WordPress Git remote + permissions + revisions

## Portable agent skills.

Push MD can export agent guidance through WordPress content guidelines. The instructions your WordPress agent uses can appear in the coding-agent checkout, so edits carry the same expectations about blocks, paths, permissions, and safe operations.

portable skill files

AGENTS.md and SKILL.md

Generated guidance lands beside Markdown, templates, and Global Styles so coding agents inherit the same site rules.

## Markdown edits, WordPress publishing.

If you like Astro, Jekyll, or a Markdown vault, Push MD gives you a familiar file tree while WordPress keeps handling previews, publishing, permissions, rendering, and the front end.

Path What WordPress does
post/*.md Turns Markdown into posts and revisions
page/**/*.md Preserves page hierarchy and identity
wp_template_part/**/*.html Keeps block markup Site Editor friendly
wp_global_styles/*.json Applies editable Global Styles overlays

## Clone WordPress like a Git remote.

Install Push MD, authenticate with a WordPress Application Password, then clone the trunk branch from your site. Your Git client talks to WordPress directly.

  1. Install and activate the Push MD plugin.
  2. Create or use a WordPress Application Password.
  3. Clone the WordPress Git remote and work on trunk.
git clone https://example.com/wp-json/git/v1/md.git my-site

Pushes to other branches are rejected. WordPress remains the source of truth.

Push MD is part of the WordPress PHP Toolkit.
