WordPress-native Git for coding agents

Let coding agents work in Git. Keep WordPress in charge.

Push MD turns WordPress into a Git remote for supported content, so agents and Markdown-first teams can clone, diff, commit, and push while WordPress remains the source of truth.

No sync layer

WordPress is the remote.

Agents and editors clone directly from /wp-json/git/v1/md.git instead of pushing through another canonical repository.

No new users

Use WordPress roles.

Git over HTTPS uses WordPress authentication, including Application Passwords, and respects existing capabilities.

Revision-backed

Pushes create revisions.

Accepted content changes go through WordPress APIs, so publishing behavior, permissions, and revisions remain intact.

Admin-safe

Pull admin edits first.

WP-Admin edits become Git-visible on pull, and stale pushes are rejected before WordPress content is changed.

One authority, many workers

Use WordPress as the sync engine between agents.

Push MD gives every authenticated worker the same Git-shaped view of supported content. Coding agents, local editors, and WP-Admin all coordinate through WordPress, so there is no second source of truth drifting out of date.

Install and clone

Start with the tools every agent already understands.

Install Push MD, authenticate with a WordPress Application Password, then clone the trunk branch from your site. Your Git client talks to WordPress directly.

  1. Install and activate the Push MD plugin.
  2. Create or use a WordPress Application Password.
  3. Clone the WordPress Git remote and work on trunk.
git clone https://example.com/wp-json/git/v1/md.git my-site

Pushes to other branches are rejected. WordPress remains the source of truth.

Push MD is part of the WordPress PHP Toolkit.
